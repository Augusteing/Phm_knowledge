# 实验1: 基线实验 (Baseline Experiment)

## 📋 实验信息

- **实验编号**: EXP-01
- **实验名称**: 基线实验
- **开始日期**: 2025-09-01
- **完成日期**: 2025-09-10
- **状态**: ✅ 已完成

## 🎯 实验目标

建立知识图谱构建的基线方法，测试基本的实体关系抽取能力，为后续实验提供对比基准。

## 🔬 实验方法

### 使用模型
- **主模型**: Gemini-2.5-pro
- **调用方式**: OpenAI兼容API

### 数据集
- **来源**: 航空领域学术论文
- **数量**: 约100篇
- **格式**: PDF → Markdown

### 抽取流程
1. 使用MinerU将PDF转换为Markdown
2. 使用基础提示词调用LLM抽取实体和关系
3. 输出JSON格式结果
4. 基本格式化处理

### 提示词策略
- 简单直接的指令
- 要求输出结构化JSON
- 包含实体类型和关系类型定义

## 📊 实验结果

### 抽取统计
- **实体总数**: 约5,000个
- **关系总数**: 约3,000个
- **平均每篇**: 实体50个，关系30个
- **千字密度**: 实体3.2个/千字，关系1.8个/千字

### 主要发现
1. ✅ 基本抽取功能可行
2. ✅ 结构化输出基本正确
3. ⚠️ 抽取结果不够稳定
4. ⚠️ 缺少元数据信息
5. ⚠️ 关系抽取不够全面

## 🔍 评估指标

### 实体关系密度
- **计算方法**: (实体/关系数量 / 文本字数) × 1000
- **实体密度**: 3.2个/千字
- **关系密度**: 1.8个/千字

### 模型打分
- **评分模型**: Gemini-2.5-pro
- **平均得分**: 7.5/10
- **评分维度**: 准确性、完整性、合理性

## ❌ 已知问题

1. **一致性问题**: 同一文献多次抽取结果差异较大（一致性约60%）
2. **元数据缺失**: 没有自动填充论文元数据（作者、单位等）
3. **关系不全面**: 某些隐含关系未被识别
4. **格式不规范**: 部分输出格式需要额外处理

## 💡 改进方向

基于本实验发现的问题，后续实验将重点改进：

1. **优化提示词**: 提供更详细的指导和示例
2. **增加元数据填充**: 自动识别和填充论文元数据
3. **多轮抽取**: 使用一致性机制提升稳定性
4. **格式规范化**: 增加后处理步骤统一格式

这些改进将在EXP-02中实施。
